# 2026-05-19

- Feishu streaming-card superpowers quick input can hit the session-mismatch confirmation card when the card-bound session differs from the current active session. In that branch, the confirmation buttons must carry the already-expanded superpowers prompt, not the action id `submit_superpowers_quick_input`; otherwise clicking `改为当前会话` loses the typed input and falls back to `请输入命令`.
- For backward compatibility, `FeishuCardActionService.HandleSuperpowersQuickActionAsync` should still accept both forms on confirmation callbacks:
  - legacy action ids such as `continue_superpowers` or `execute_superpowers_plan`
  - fully expanded raw prompts produced from the original card input
