# 表单容器（form）

表单容器支持用户在前端本地录入一批表单项后，通过点击一次提交按钮，将这一批本地缓存的表单内容一次回调至开发者的服务端，实现异步提交多个表单项数据的效果。

## 注意事项

- 表单容器仅支持卡片 JSON 2.0 结构
- 表单容器内的所有表单项数据会在用户本地缓存，直到用户点击提交按钮才会一次性提交
- 每个卡片最多支持 1 个表单容器
- 表单容器内不支持嵌套表单容器
- 表单容器内不支持内嵌表格组件

## 应用场景

- 问卷调查：批量收集用户反馈、评价等信息
- 信息收集：收集用户的联系方式、地址等结构化信息
- 工单提交：支持用户填写多个字段提交工单
- 审批申请：集中收集审批所需的各项信息

## 组件结构

表单容器包含三部分：
1. 表单容器本身（form）：作为所有表单项的外层容器
2. 表单项组件：输入框、选择器、日期选择器等交互组件
3. 提交按钮（button）：触发表单提交的按钮，必须放在表单容器内

## JSON 结构

```json
{
  "schema": "2.0",
  "body": {
    "elements": [
      {
        "tag": "form",
        "name": "form_name", // 表单标识，提交回调时会返回
        "elements": [
          // 表单项
          {
            "tag": "input",
            "name": "username", // 表单项标识，提交回调时会返回
            "placeholder": {
              "tag": "plain_text",
              "content": "请输入用户名"
            }
          },
          {
            "tag": "input",
            "name": "email",
            "placeholder": {
              "tag": "plain_text",
              "content": "请输入邮箱"
            }
          },
          // 提交按钮
          {
            "tag": "button",
            "text": {
              "tag": "plain_text",
              "content": "提交"
            },
            "type": "primary",
            "action": {
              "type": "form_submit" // 按钮类型必须为 form_submit
            }
          }
        ]
      }
    ]
  }
}
```

## 字段说明

| 字段 | 必填 | 类型 | 默认值 | 描述 |
|------|------|------|--------|------|
| tag | 是 | String | / | 组件标签，固定值为 `form` |
| name | 是 | String | / | 表单的唯一标识，提交回调时会返回该字段，用于区分不同表单 |
| element_id | 否 | String | / | 操作组件的唯一标识，用于组件级别的更新操作 |
| margin | 否 | String | 0px | 表单容器的外边距，取值范围 [-99,99]px |
| padding | 否 | String | 0px | 表单容器的内边距，取值范围 [0,99]px |
| vertical_spacing | 否 | String | 12px | 表单容器内组件的垂直间距，可选值：small(4px)、medium(8px)、large(12px)、extra_large(16px) 或具体像素值 |
| horizontal_spacing | 否 | String | 12px | 表单容器内组件的水平间距，可选值同上 |
| background_style | 否 | String | default | 表单容器的背景色样式 |
| elements | 是 | Array | / | 表单容器内的组件列表，包含表单项和提交按钮 |

## 表单提交回调

当用户点击提交按钮时，飞书服务器会向开发者的回调地址发送回调请求，包含所有表单项的填写值：

```json
{
  "schema": "2.0",
  "event_type": "card.action.trigger",
  "action": {
    "value": {},
    "tag": "button",
    "form_value": {
      "username": "张三", // 表单内 input 组件的 name 作为 key，用户输入作为 value
      "email": "zhangsan@example.com"
    },
    "form_name": "form_name" // 表单容器的 name 字段
  }
}
```

## 最佳实践

1. **表单项命名**：为每个表单项设置清晰的 `name` 属性，方便后端解析
2. **必填项校验**：前端暂不支持必填校验，建议在服务端对提交的数据进行校验
3. **提交反馈**：用户提交后，建议返回 toast 提示提交结果，并可以更新卡片为提交成功状态
4. **表单重置**：如果需要重置表单，可以在提交成功后全量更新卡片，重新渲染空白表单
