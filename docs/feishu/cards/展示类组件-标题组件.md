# 标题（title）

卡片的标题组件支持添加卡片主标题、副标题、后缀标签和标题图标。

本文档介绍标题组件的 JSON 2.0 结构，要查看历史 JSON 1.0 结构，参考标题。

## 注意事项

同一张卡片仅支持添加一个标题组件。

## 组件属性

### JSON 结构

标题组件的完整 JSON 2.0 结构如下所示：

```json
{
  "schema": "2.0",
  "header": {
    "title": {
      "tag": "plain_text",
      "content": "示例标题"
    },
    "subtitle": {
      "tag": "plain_text",
      "content": "示例文本"
    },
    "text_tag_list": [
      {
        "tag": "text_tag",
        "element_id": "custom_id",
        "text": {
          "tag": "plain_text",
          "content": "标签 1"
        },
        "color": "neutral"
      }
    ],
    "i18n_text_tag_list": {
      "zh_cn": [],
      "en_us": [],
      "ja_jp": [],
      "zh_hk": [],
      "zh_tw": []
    },
    "template": "blue",
    "icon": {
      "tag": "standard_icon",
      "token": "chat-forbidden_outlined",
      "color": "orange",
      "img_key": "img_v2_38811724"
    },
    "padding": "12px 8px 12px 8px"
  }
}
```

### 字段说明

| 名称 | 必填 | 类型 | 默认值 | 描述 |
|------|------|------|--------|------|
| title | 是 | Object | / | 配置卡片的主标题信息。注意：如果只配置副标题，则实际展示为主标题效果。 |
| └ tag | 是 | String | / | 文本类型的标签。可取值：<br>- `plain_text`：普通文本内容或表情<br>- `lark_md`：支持 Markdown 语法的文本内容 |
| └ content | 否 | String | 空 | 卡片主标题内容。要为标题配置多语言，参考配置卡片多语言。注意：主标题内容最多四行，超出四行的内容用 ... 省略。 |
| subtitle | 否 | Object | / | 配置卡片的副标题信息。注意：如果只配置副标题，则实际展示为主标题效果。 |
| └ tag | 是 | String | / | 文本类型的标签。可取值同 title.tag |
| └ content | 否 | String | 空 | 卡片副标题内容。要为标题配置多语言，参考配置卡片多语言。注意：副标题内容最多一行，超出一行的内容用 ... 省略。 |
| text_tag_list | 否 | TextTagList[] | [] | 添加标题的后缀标签。最多可添加 3 个标签内容，如果配置的标签数量超过 3 个，则取前 3 个标签进行展示。标签展示顺序与数组顺序一致。注意：`text_tag_list` 和 `i18n_text_tag_list` 只能配置其中之一。如果同时配置仅生效 `i18n_text_tag_list`。 |
| └ tag | 是 | String | / | 后缀标签的标识。固定取值：`text_tag` |
| └ element_id | 否 | String | 空 | 操作组件的唯一标识。JSON 2.0 新增属性。用于在调用组件相关接口中指定组件。在同一张卡片内，该字段的值全局唯一。仅允许使用字母、数字和下划线，必须以字母开头，不得超过 20 字符。 |
| └ text | 否 | Text Object | / | 后缀标签的内容。基于文本组件的 plain_text 模式定义内容。 |
| └ color | 否 | String | `blue` | 后缀标签的颜色，可选值参见下文的后缀标签颜色枚举。 |
| i18n_text_tag_list | 否 | Object | / | 配置后缀标签的多语言属性，在所需语种字段下添加完整的后缀标签结构体即可。每个语言最多可配置 3 个标签内容，如果配置的标签数量超过 3 个，则取前 3 个标签进行展示。标签展示顺序与数组顺序一致。支持的多语言枚举值：<br>- `zh_cn`：简体中文<br>- `en_us`：英文<br>- `ja_jp`：日文<br>- `zh_hk`：繁体中文（中国香港）<br>- `zh_tw`：繁体中文（中国台湾）<br>- `id_id`：印尼语<br>- `vi_vn`：越南语<br>- `th_th`：泰语<br>- `pt_br`：葡萄牙语<br>- `es_es`：西班牙语<br>- `ko_kr`：韩语<br>- `de_de`：德语<br>- `fr_fr`：法语<br>- `it_it`：意大利语<br>- `ru_ru`：俄语<br>- `ms_my`：马来语<br>注意：`text_tag_list` 和 `i18n_text_tag_list` 只能配置其中之一。如果同时配置两个字段，则优先生效多语言配置。 |
| template | 否 | String | `default` | 配置标题主题颜色。可选值参见下文的标题主题样式枚举。 |
| icon | 否 | Object | / | 添加图标作为文本前缀图标。支持自定义或使用图标库中的图标。 |
| └ tag | 否 | String | / | 图标类型的标签。可取值：<br>- `standard_icon`：使用图标库中的图标<br>- `custom_icon`：使用自定义图片作为图标 |
| └ token | 否 | String | / | 图标库中图标的 token。当 tag 为 `standard_icon` 时生效。枚举值参见图标库。 |
| └ color | 否 | String | / | 图标的颜色。支持设置线性和面性图标（即 token 末尾为 outlined 或 filled 的图标）的颜色。当 tag 为 `standard_icon` 时生效。枚举值参见颜色枚举值。 |
| └ img_key | 否 | String | / | 自定义前缀图标的图片 key。当 tag 为 `custom_icon` 时生效。 |
| padding | 否 | String | `12px` | 标题组件的内边距。值的取值范围为 [-99,99]px。可选值格式：<br>- 单值，如 `"10px"`，表示容器的四个内边距都为 10 px<br>- 双值，如 `"4px 0"`，表示容器的上下内边距为 4 px，左右内边距为 0 px<br>- 多值，如 `"4px 0 4px 0"`，表示容器的上、右、下、左的内边距分别为 4px，0，4px，0 |

## Demo 示例

以下 JSON 2.0 结构的示例代码可实现带有主副标题、多个标签、图标和蓝色主题的标题效果：

```json
{
  "schema": "2.0",
  "body": {
    "elements": [
      {
        "tag": "div",
        "text": {
          "tag": "plain_text",
          "content": "示例文本"
        }
      }
    ]
  },
  "header": {
    "title": {
      "tag": "lark_md",
      "content": ":Partying:卡片主标题 "
    },
    "subtitle": {
      "tag": "plain_text",
      "content": "卡片副标题"
    },
    "text_tag_list": [
      {
        "tag": "text_tag",
        "text": {
          "tag": "plain_text",
          "content": "标签 1"
        },
        "color": "blue"
      },
      {
        "tag": "text_tag",
        "text": {
          "tag": "plain_text",
          "content": "标签 2"
        },
        "color": "turquoise"
      },
      {
        "tag": "text_tag",
        "text": {
          "tag": "plain_text",
          "content": "标签 3"
        },
        "color": "orange"
      }
    ],
    "template": "blue",
    "icon": {
      "tag": "standard_icon",
      "token": "larkcommunity_colorful"
    },
    "padding": "12px"
  }
}
```

## 枚举

### 标题主题样式枚举

标题组件中的 template 字段决定了卡片的标题主题样式：

| 枚举值 | 描述 |
|--------|------|
| `blue` | 蓝色主题 |
| `wathet` | 浅蓝色主题 |
| `turquoise` | 绿松石色主题 |
| `green` | 绿色主题 |
| `yellow` | 黄色主题 |
| `orange` | 橙色主题 |
| `red` | 红色主题 |
| `carmine` | 深红色主题 |
| `violet` | 紫罗兰色主题 |
| `purple` | 紫色主题 |
| `indigo` | 靛蓝色主题 |
| `grey` | 灰色主题 |
| `default` | 默认主题 |

### 后缀标签颜色枚举

标题的后缀标签的颜色样式由 color 字段定义：

| 枚举值 | 描述 |
|--------|------|
| `neutral` | 中性色 |
| `blue` | 蓝色 |
| `turquoise` | 绿松石色 |
| `lime` | 酸橙色 |
| `orange` | 橙色 |
| `violet` | 紫罗兰色 |
| `indigo` | 靛蓝色 |
| `wathet` | 浅蓝色 |
| `green` | 绿色 |
| `yellow` | 黄色 |
| `red` | 红色 |
| `carmine` | 深红色 |
| `purple` | 紫色 |

### 标题主题样式建议

在群聊中，可使用彩色标题。对于需高亮提醒的卡片信息，可将标题配置为应用的品牌色或表达状态的语义色，增强信息的视觉锚点。在单聊中，建议根据卡片的状态配置标题样式：

| 样式颜色 | 语义 |
|----------|------|
| 绿色（green） | 完成或成功 |
| 橙色（orange） | 警告或警示 |
| 红色（red） | 错误或异常 |
| 灰色（grey） | 失效 |
