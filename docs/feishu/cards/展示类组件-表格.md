# 表格（table）

飞书卡片支持表格组件，并支持在表格中添加普通文本、富文本、选项标签、数字、人员列表、日期类型的内容。

本文档介绍表格组件的 JSON 2.0 结构，要查看历史 JSON 1.0 结构，参考表格。

## 注意事项

- 单张卡片最多支持放置五个表格组件。若卡片配置了多语言，则单个语言最多支持放置五个表格组件。
- 当单元格内剩余空间无法完整展示内容时，末尾将省略。在客户端，用户可通过光标悬浮或点击的方式查看被省略的内容。

## 嵌套规则

- 表格组件不可被内嵌在其它组件内，只可放在卡片根节点下。
- 表格组件不支持内嵌其它组件。

## JSON 结构

表格组件的完整 JSON 2.0 结构如下所示：

```json
{
  "schema": "2.0",
  "body": {
    "elements": [
      {
        "tag": "table",
        "element_id": "custom_id",
        "margin": "0px 0px 0px 0px",
        "page_size": 5,
        "row_height": "low",
        "row_max_height": "50px",
        "freeze_first_column": true,
        "header_style": {
          "text_align": "left",
          "text_size": "normal",
          "background_style": "none",
          "text_color": "grey",
          "bold": true,
          "lines": 1
        },
        "columns": [
          {
            "name": "customer_name",
            "display_name": "客户名称",
            "width": "auto",
            "data_type": "text",
            "vertical_align": "top",
            "horizontal_align": "left"
          },
          {
            "name": "customer_arr",
            "display_name": "ARR(万元)",
            "data_type": "number",
            "format": {
              "symbol": "¥",
              "precision": 2,
              "separator": true
            },
            "width": "120px"
          },
          {
            "name": "customer_scale",
            "display_name": "客户规模",
            "data_type": "options"
          },
          {
            "name": "customer_poc",
            "display_name": "客户对接人",
            "data_type": "persons"
          },
          {
            "name": "meeting_date",
            "display_name": "对接时间",
            "data_type": "date",
            "date_format": "YYYY/MM/DD"
          }
        ],
        "rows": [
          {
            "customer_name": "飞书科技",
            "customer_date": 1699341315000,
            "customer_scale": [
              {
                "text": "S2",
                "color": "blue"
              }
            ],
            "customer_arr": 168,
            "customer_poc": [
              "ou_14a32f1a02e64944cf19207aa43abcef",
              "ou_e393cf9c22e6e617a4332210d2aabcef"
            ]
          }
        ]
      }
    ]
  }
}
```

## 字段说明

| 名称 | 必填 | 类型 | 默认值 | 描述 |
|------|------|------|--------|------|
| tag | 是 | String | / | 组件的标签。表格组件的固定取值为 `table` |
| element_id | 否 | String | 空 | 操作组件的唯一标识。JSON 2.0 新增属性。用于在调用组件相关接口中指定组件。在同一张卡片内，该字段的值全局唯一。仅允许使用字母、数字和下划线，必须以字母开头，不得超过 20 字符。 |
| margin | 否 | String | `0` | 组件的外边距。值的取值范围为 [-99,99]px。可选值格式：<br>- 单值，如 `"10px"`，表示组件的四个外边距都为 10 px<br>- 双值，如 `"4px 0"`，表示组件的上下外边距为 4 px，左右外边距为 0 px<br>- 多值，如 `"4px 0 4px 0"`，表示组件的上、右、下、左的外边距分别为 4px，0，4px，0 |
| page_size | 否 | Number | `5` | 每页最大展示的数据行数。支持 [1,10] 整数。 |
| row_height | 否 | String | `low` | 表格的行高。单元格高度如无法展示一整行内容，则上下裁剪内容。可取值：<br>- `low`：低<br>- `middle`：中<br>- `high`：高<br>- `auto`：行高自适应内容。JSON 2.0 新增枚举，V7.33 及以上客户端版本支持<br>- `[32,124]px`：自定义行高，单位为像素，如 40px。取值范围是 [32,124] |
| row_max_height | 否 | String | `124px` | 当 row_height 为 auto 时，可使用该参数设置最大行高。若内容超过该值，将被裁剪。取值范围为 [32,999]，单位为像素。JSON 2.0 新增属性，V7.33 及以上客户端版本支持。 |
| freeze_first_column | 否 | Boolean | `false` | 是否冻结首列：<br>- `true`：冻结首列。即左右滚动表格时不滚动首列，其余列叠加展示在首列底下<br>- `false`：不冻结首列。即左右滚动表格时所有表格均做滚动 |
| header_style | 否 | Object | / | 表头样式风格。详见下方 header_style 字段说明。 |
| columns | 是 | Array | [] | 列对象数组。详见下方 column 字段说明。 |
| rows | 是 | Array | [] | 行对象数组。与列定义对应的数据。用 "name":VALUE 的形式，定义每一行的数据内容。name即你自定义的列标记。 |

### header_style 字段说明

| 字段 | 是否必填 | 类型 | 默认值 | 描述 |
|------|----------|------|--------|------|
| text_align | 否 | String | `left` | 表头文本对齐方式：`left`、`center`、`right` |
| text_size | 否 | String | `normal` | 表头文本大小：`normal`(正文14px)、`heading`(标题16px) |
| background_style | 否 | String | `none` | 表头背景色：`grey`、`none` |
| text_color | 否 | String | `default` | 文本颜色：`default`(根据主题自动适配)、`grey` |
| bold | 否 | Boolean | `true` | 表头文本是否加粗 |
| lines | 否 | Number | `1` | 表头文本的行数。支持大于等于 1 的整数。 |

### column 字段说明

| 字段 | 是否必填 | 类型 | 默认值 | 描述 |
|------|----------|------|--------|------|
| name | 是 | String | 空 | 列的 key（键名）。用于在行数据对象数组中，指定数据填充的单元格。 |
| display_name | 否 | String | 空 | 在表头展示的列名称。不填或为空则不展示列名称。 |
| width | 否 | String | `auto` | 列宽度：<br>- `auto`：自适应内容宽度<br>- 自定义宽度：如 `120px`，取值范围是 [80px,600px] 的整数<br>- 自定义宽度百分比：如 `25%`，取值范围是 [1%,100%] |
| vertical_align | 否 | String | `center` | 列内数据垂直对齐方式：`top`、`center`、`bottom` |
| horizontal_align | 否 | String | 数字类型默认为 `right`，其他为 `left` | 列内数据水平对齐方式：`left`、`center`、`right` |
| data_type | 是 | String | `text` | 列数据类型，可选值：<br>- `text`：不带格式的普通文本<br>- `lark_md`：支持部分 markdown 格式的文本<br>- `options`：选项标签<br>- `number`：数字<br>- `persons`：人员列表<br>- `date`：日期时间<br>- `markdown`：支持完整 Markdown 语法的文本 |
| format | 否 | Object | / | 该字段仅当 data_type 为 number 时生效：<br>- `precision`：数字的小数点位数，0~10 的整数<br>- `symbol`：数字前的货币单位，如 "¥"<br>- `separator`：是否按千分位逗号分割，默认 false |
| date_format | 否 | String | 空 | 该字段仅当 data_type 为 date 时生效。可使用占位符组合：<br>- `YYYY`：年<br>- `MM`：月<br>- `DD`：日<br>- `HH`：小时<br>- `mm`：分钟<br>- `ss`：秒<br>推荐格式：`YYYY/MM/DD`、`YYYY-MM-DD HH:mm` 等 |
